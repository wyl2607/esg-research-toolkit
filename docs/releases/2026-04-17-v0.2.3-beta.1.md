# ESG Research Toolkit v0.2.3-beta.1 Release

**Release Date**: 2026-04-17
**Previous Version**: v0.2.2 (2026-04-17)
**Status**: Beta

## Overview

This beta release brings the repository back into a professional, internally
consistent state after a fast sequence of backend hardening and UI work. The
main goals are:

- align runtime version metadata with release documentation
- harden model configuration and health visibility
- tighten report/framework data-integrity controls
- keep local-only archive material out of the public GitHub surface

This is intentionally marked as a beta rather than a final stable release.
The code has moved forward, but the team still needs one more pass on release
metadata discipline and post-push security hygiene before cutting a new stable
tag.

## Highlights

- Centralized model registry in `core/models.py`
- New `GET /health/models` endpoint for operational visibility
- Stronger uniqueness / dedup controls in report and framework storage
- `report_parser` route split for lower file complexity and safer future edits
- Version alignment to `v0.2.3-beta.1`
- Local archive artifacts excluded from GitHub by default

## Why Beta

The repository is in a healthier state than the previous published snapshot,
but this release is still treated as a beta for two reasons:

1. recent release metadata had drifted (`main.py` vs `docs/releases/VERSION.md`)
2. GitHub push feedback still reports dependency vulnerabilities that should be
   triaged before a stable label is used

## Key Changes

### Backend Hardening

- `core/models.py` now defines extraction / validation / audit model purposes,
  default models, max-token settings, fallback order, and availability checks.
- `main.py` exposes the model-health payload and now reports the aligned app
  version string.
- `report_parser/storage.py` and `esg_frameworks/storage.py` strengthen
  integrity controls around deduplication and uniqueness.

### API Structure

- `report_parser/admin_routes.py` now owns deletion/export management routes.
- `report_parser/industry_routes.py` now owns the by-industry query route.
- `report_parser/api.py`, `taxonomy_scorer/api.py`, and `esg_frameworks/api.py`
  add stricter response-model annotations for contract safety.

### Repo Hygiene

- `.gitignore` now excludes `docs/archive/`, keeping local archive notes out of
  the public repository by default.
- Local archive material that previously triggered security/push guard issues
  was removed from the GitHub-facing change surface.
