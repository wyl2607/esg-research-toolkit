# v0.3.1 Release Notes - 2026-04-24

## Summary

This patch release focused on public CI convergence, production safety defaults, contract hardening, and repository hygiene after a parallel PR stabilization loop.

## Added

- Production startup now requires the Alembic-managed path in production-oriented configuration, with guardrail documentation for operators.
- Core module unit coverage was expanded for AI client behavior and limiter reset isolation.
- OpenAPI contract tests now cover additional production-safety and request-parse failure surfaces.

## Changed

- Production defaults and deployment workflows were tightened to reduce unsafe implicit startup paths.
- Public repository hygiene rules were updated so local-only artifacts and scratch files stay outside the GitHub surface.
- Frontend generated API types were regenerated against the CI-compatible toolchain to keep contract drift visible and reproducible.
- PostCSS dev dependency was updated through Dependabot.

## Fixed

- JSON request parse failure responses are now explicitly documented as `400` responses in the API contract.
- Regex fallback unit normalization was corrected and merged into the main parser flow.
- The GitHub branch protection required check was aligned to the actual Actions check run name `leak-guard`, unblocking otherwise-green PRs.
- CI contract runs were stabilized against Schemathesis v4 CLI behavior and generated type drift.

## Verified

- Latest `main` head: `e650838f3ac629715dbb94d414b03634a9de2d9f`.
- GitHub Actions on latest `main`: `security-guard`, `Lint`, `API Contracts`, and `Tests` all succeeded.
- `Deploy to VPS` was skipped for the latest main run; this was not a failure.
- Open PR queue was empty after PRs #14, #16, #19, #22, #24, and #25 merged or were superseded.

## Operational Notes

- The local main worktree may still contain local-only artifact classification edits from earlier cleanup work; use a clean branch or isolated worktree for public changes.
- Temporary `/tmp` worktrees may need `CONVERGE_WORKTREE_GUARD=0` for push verification because the converge guard expects the canonical repository path. Zone and security guards must still run.
